## 一【严重·需确认】backtest.py 的 target 对齐关系

### 核心问题

`build_dataset.py` 中 target 的定义：
```python
df["return_1d"] = df["price_close"].pct_change()        # return_1d[i] = (close[i]-close[i-1])/close[i-1]
df["target"] = np.sign(df["return_1d"].shift(-1))        # target[i] = sign(return_1d[i+1])
```

`trainer.py` 的 `get_window_data` 中：
```python
y.append(sub["target"].iloc[i])           # target[i] = sign(次日收益)
ret.append(sub["return_1d_orig"].iloc[i]) # return_1d_orig[i] = 当日收益
```

### 对齐分析

- `signals[t]` 对应某行 R，预测 `target[R]` = sign(close[R+1] - close[R]) → **R→R+1 的方向**
- `returns[t]` 对应同一行 R，值为 `return_1d_orig[R]` = (close[R] - close[R-1])/close[R-1] → **R-1→R 的收益**

backtest.py 当前代码：
```python
pos = 1 if signals[t-1] >= 0.5 else 0   # signals[t-1] 对应行 R_{t-1}，预测 R_{t-1}→R_t 的方向
net_returns[t] = pos * returns[t]         # returns[t] 对应行 R_t，值为 R_{t-1}→R_t 的收益
```

**结论：当前 backtest 的 `signals[t-1] × returns[t]` 配对是正确的。**
- signals[t-1] 预测的是"R_{t-1}→R_t 这段时间的方向"
- returns[t] 恰好是"R_{t-1}→R_t 这段时间的收益"
- 两者对应同一时间段，配对正确。

原始代码（修复前）的 `signals[t] × returns[t]` 才是错的：signals[t] 预测 R_t→R_{t+1} 方向，但 returns[t] 是 R_{t-1}→R_t 收益，差了一天。

## 二【严重·已修复】regime_final.py 的 prep_df ffill 冻结 bug

**问题**：用训练期 close 算 SMA20/50 和 RSI，然后 `.reindex().ffill()` 到测试期。
训练期结束后 rolling 值变为 NaN，最后一个有效值被 ffill 到整个测试期。
导致测试期 `sma20`、`sma50`、`rsi14` 全部为常数。

**修复方案**：改用滚动窗口计算，对每个时间点只使用该点之前的数据。
详见 `regime_final.py` 和 `continuous_all.py` 的 `prep_df()` / `prep()` 函数。

---

## 三【严重·已修复】FOLDS 编号与论文不一致

**问题**：代码中 FOLDS 不是时间单调递增的（Fold 1=2022, Fold 5=2018），
论文声称"时间单调前推的 walk-forward"，但代码输出的 fold 标签与论文表不对应。

**修复方案**：将所有文件的 FOLDS 统一为时间单调递增顺序。

---

## 四【严重·已修复】SMA 基线计算口径不统一

**问题**：三个文件用了三套 SMA 计算方式：
- `train.py`：用 train+test combined 算 SMA
- `analyze_all.py`：用全量数据算 SMA（rolling 不看未来，不算泄漏，但口径不同）
- `regime_final.py`：只用训练期算 SMA 再 ffill（冻结 bug）

**修复方案**：统一为 `analyze_all.py` 的方式（全量 rolling），因为 SMA 是滚动指标，t 时刻只用过去值，不存在未来泄漏。

---

## 五【中等·已修复】regime 阈值不一致

**问题**：
- `regime_final.py`：`vol_p75=0.038`，crisis 触发 = `v > 0.038*2 = 0.076`
- `continuous_all.py`：直接写 `v > 0.076`
- 论文声称统一为 0.093

**修复方案**：统一为 0.093（训练段校准值），并更新代码中的阈值。

---

## 六【中等·已修复】run_ablation.py 的 FOLDS.index 崩溃 bug

**问题**：`REGIMES[FOLDS.index((fname,))]` 永远找不到（FOLDS 元素是 5-tuple，不是 1-tuple），
会抛 ValueError。说明 summary 代码从未成功运行。

**修复方案**：改用枚举索引。

---

## 七【中等·已修复】DSR 的 N_trials=3 严重低估

**问题**：实际试验次数（binary/continuous、frozen/tuned、full/price-only、多组阈值、多种架构）
远超 3。N 填 3 等于几乎不惩罚，DSR 显著性结论偏乐观。

**修复方案**：将 N_trials 改为更合理的值（如 20）。
